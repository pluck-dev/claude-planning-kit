# /pencil-check - 디자인/코드 상태 누락 체크

현재 작업 중인 페이지 또는 컴포넌트에서 빠진 상태를 찾아 리포트한다.

## 참조 스킬
`~/.claude/skills/pencil-design-system.md` 섹션 4 (상태 보강 체크리스트) 적용.

## 수행 절차

### 1. 대상 파악
- .pen 파일이면 → 캔버스에 어떤 상태가 있는지 확인
- .tsx/.jsx 파일이면 → 코드에서 처리된 상태 확인
- 지정 없으면 → 최근 작업 파일 대상

### 2. 컴포넌트 타입 분류
- 인터랙티브 (버튼, 토글 등) → 인터랙션 상태 6종 체크
- 데이터 표시 (리스트, 카드 등) → 데이터 상태 8종 체크
- 폼 (입력, 선택 등) → 폼 상태 10종 체크
- 페이지 → 페이지 상태 + 반응형 체크

### 3. 리포트 출력

```
🔍 [컴포넌트명] 상태 체크

✅ 있음: Default, Hover, With Data
❌ 없음: Empty State, Loading, Error, Disabled
⚠️ 권장: Overflow(텍스트 말줄임), Single Item 레이아웃

💡 제안:
- Empty: "데이터가 없습니다" + 등록 버튼
- Loading: 스켈레톤 3행
- Error: "불러오기 실패" + 재시도
```

### 4. 자동 보강
사용자가 원하면 빠진 상태를 .pen 캔버스에 추가하거나 코드에 반영한다.
