# /pencil-init - 디자인 시스템 초기화

프로젝트에 Pencil.dev 기반 디자인 폴더를 생성합니다.

## 참조 스킬
반드시 `~/.claude/skills/pencil-design-system.md`를 읽고 적용합니다.

## 사용법
```
/pencil-init
```

## 수행 절차

### Phase 1: 프로젝트 정보 수집

1. 사용자에게 확인:
   - **프로젝트 이름** (폴더명 + .pen 파일명으로 사용)
   - 주요 화면/플로우는 뭔가요? (랜딩+서비스, 인증+대시보드 등)
   - 디자인 토큰 기준이 있나요? (피그마 등)
   - 코드 프로젝트도 함께 세팅할 건가요? (디자인만 / 디자인+코드)

2. 현재 프로젝트 구조 탐색:
   - 기존 디자인 폴더가 있는지 확인
   - 기존 `src/` 구조 파악

### Phase 2: 폴더 구조 생성

**핵심 원칙: .pen 파일은 하나로 시작한다.**

처음에는 모든 디자인을 하나의 캔버스에서 작업하고,
나중에 컴포넌트를 분리할 때 `_system/` 폴더를 생성한다.

```
{프로젝트명}/
  {프로젝트명}.pen    ← 메인 디자인 파일 (여기서 전부 작업)
  _system/            ← (나중에) 컴포넌트 분리 시 생성
    primitives/       ← 버튼, 인풋 등 원자 컴포넌트
    composites/       ← 카드, 모달 등 조합 컴포넌트
    tokens/           ← 색상, 타이포 등 디자인 토큰
  README.md           ← 디자인 맵 + 상태 체크리스트
```

- `_system/` 폴더는 Phase 2에서 생성하지 않음
- `{프로젝트명}.pen` 하나와 `README.md`만 생성

### Phase 3: README 생성

```markdown
# {프로젝트명} - Design System

## 구조
{프로젝트명}/
  {프로젝트명}.pen  ← 메인 디자인 파일
  _system/          ← (나중에) 컴포넌트 분리 시 생성
  README.md

## 워크플로우
[1] {프로젝트명}.pen에서 전체 페이지 디자인
    ↓
[2] 반복 패턴 발견 → 컴포넌트 식별
    ↓
[3] /pencil-spec [컴포넌트명] → _system/에 분리 + 상태 보강
    ↓
[4] 코드 전환

## 화면 구성
- (사용자 답변 기반으로 작성)

## 페이지 디자인 시 체크
- [ ] Empty State (데이터 0건)
- [ ] Loading State (스켈레톤)
- [ ] Error State (재시도 버튼)
- [ ] Overflow (텍스트 말줄임)
- [ ] 반응형 (Mobile + Desktop)
```

## 중요 원칙
- **하나의 .pen 파일로 시작** — 빈 파일 여러 개 만들지 않음
- 폴더명은 프로젝트 이름으로 통일
- `_system/` 폴더는 컴포넌트 분리 단계에서 생성
- 기존 프로젝트 구조를 존중하고 덮어쓰지 않음
