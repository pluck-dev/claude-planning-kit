## 상태 워크플로우

모든 문서는 아래 상태를 따릅니다:

```
[draft] → [in-review] → [approved]
               ↓              ↓
        [revision-needed] ←──┘
                              ↓
                       [deprecated]
```

| 상태 | 설명 |
|------|------|
| `draft` | 초안 작성 중 |
| `in-review` | 리뷰/검토 중 |
| `approved` | 승인 완료 |
| `revision-needed` | 수정 필요 (리뷰 후 반려) |
| `deprecated` | 폐기됨 (더 이상 유효하지 않음) |

## 버전 규칙 (Semantic Versioning)

| 버전 범위 | 의미 | 예시 |
|-----------|------|------|
| `0.x.x` | 초안 (draft 상태) | 0.1.0, 0.2.0 |
| `1.0.0` | 최초 승인 (approved) | 1.0.0 |
| `1.x.0` | 승인 후 내용 변경 (minor) | 1.1.0, 1.2.0 |
| `x.x.x` (patch) | 오타/서식 수정 | 1.0.1, 1.1.1 |
| `2.0.0+` | 구조 완전 변경 (major) | 2.0.0 |

## Frontmatter 공통 필드

모든 문서의 frontmatter에 포함되는 필드:

```yaml
---
id: "PREFIX-NNN"        # 문서 고유 ID
title: ""               # 문서 제목
type: ""                # 문서 타입 (prd, screen-design, api-spec 등)
status: "draft"         # draft | in-review | approved | revision-needed | deprecated
version: "0.1.0"       # Semantic Versioning
created: "YYYY-MM-DD"  # 생성일
updated: "YYYY-MM-DD"  # 최종 수정일
author: ""              # 작성자
reviewer: ""            # 리뷰어
approver: ""            # 승인자
tags: []                # 분류 태그
related: []             # 관련 문서 위키링크 [[FNC-001]]
priority: "medium"      # critical | high | medium | low
phase: ""               # 프로젝트 단계
---
```

## Living Spec 시스템

> Next.js 퍼블 화면이 곧 화면설계서가 되는 인터랙티브 시스템. 코드와 Vault 문서를 자동으로 동기화합니다.

### 작동 방식

```
Vault (SCR.md) ──export──→ spec-data/*.json ──→ Next.js SpecLabel ──→ 인터랙티브 라벨
     ↑                                                                         │
     └─────────────────── import (스크린샷 + 매핑 테이블) ←─────────────────────┘
```

### CLI 명령어

| 명령어 | 설명 |
|--------|------|
| `npm run spec:export` | SCR.md → spec JSON 변환 |
| `npm run spec:import` | 스크린샷 캡처 → Vault 임베드 |
| `npm run spec:check` | Vault ↔ Code 정합성 리포트 |

### 모드 (Ctrl+Shift+S)

| 모드 | 설명 |
|------|------|
| Normal | 일반 화면 (라벨 숨김) |
| Spec | ①②③ 라벨 표시 + 클릭 시 상세 팝업 |
| Screenshot | 라벨만 표시 (캡처용, 인터랙션 비활성) |

### 관련 스킬

- `/vault-plan publish [screenId]` - 개별 화면 퍼블리시
- `/vault-plan publish all` - 전체 화면 퍼블리시
- `/orchestra phase 6a` ~ `6c` - Living Spec 단계별 실행

## 시작 가이드

### 새 문서 만들기
1. Templater 단축키 (`Alt+N` 또는 `Ctrl+T`) 실행
2. 원하는 템플릿 선택 (예: `tpl-기능명세서`)
3. 문서 ID, 제목, 작성자 입력 (커서 위치에서 Tab으로 이동)
4. 해당 폴더로 파일 이동 (예: `03-개발/기능명세서/FNC-001.md`)
5. `related` 필드에 관련 문서 위키링크 추가

### Templater 설정
1. **Settings → Templater → Template folder location**: `_templates`
2. **Trigger Templater on new file creation**: 활성화

### 대시보드 확인
- `05-대시보드/00-전체현황.md`에서 전체 문서 현황 확인
- Dataview 플러그인이 설치되어 있어야 정상 동작
